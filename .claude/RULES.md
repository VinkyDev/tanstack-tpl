**!important 本文规范必须遵守**

通用规范：回复内容，代码注释，UI文字等均使用中文。

通用代码规范

1. 高内聚，低耦合：对于同类逻辑，应该提供一个统一的适配层解耦，便于维护和拓展。
2. 精简注释原则：不写无价值注释，只注释关键代码。通常情况下，用更好的函数命名和结构拆分来表达代码意图，而不是写冗余的注释。
3. 不重复造轮子：尽可能的使用成熟的第三方库来实现功能。如果无法找到合适的第三方库，再考虑自己实现。如果实现的功能是通用的，那么应该考虑将其封装成utils。

TS 代码规范：

1. 类型安全：原则上不使用 any。如果不可避免使用 any ，那么应该使用 unknown 代替。
2. UI组件：尽可能的使用 shadcn 组件（或基于 shadcn 的其他 UI 库），在此基础上进行二次开发。可使用 MCP 查看可用组件。
